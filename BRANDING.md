# safe-link-checker Brand Guidelines

Welcome to the `safe-link-checker` brand guidelines. Consistency across our ecosystem helps developers instantly recognize and trust our URL intelligence tools.

## 1. Our Identity

**safe-link-checker** is the default URL security layer for the JavaScript ecosystem.
Our identity is:
- **Trustworthy**: We process sensitive security data with accuracy.
- **Modern**: We belong alongside tools like Next.js, Vercel, and Prisma.
- **Accessible**: Security shouldn't require a Ph.D. We explain everything clearly.

## 2. Logo & Mascot

### The Shield & Chain
Our primary logo represents the intersection of security (shield) and web links (chain). It should always be displayed on a dark background or with high contrast.

- **Primary Logo Path**: `apps/docs/docs/public/assets/logo.png`

### Mascot: "Sentinel" the Owl
Sentinel is our futuristic, robotic owl. Owls represent vigilance, wisdom, and the ability to see in the dark—exactly what our SDK provides for developers facing obfuscated threats.
Sentinel is used for:
- 404 pages
- Playground loading states
- Open-source swag (stickers, t-shirts)
- Major release announcements

- **Mascot Path**: `apps/docs/docs/public/assets/mascot.png`

## 3. Color Palette

Our aesthetic is "Cyber Security Neon."

| Color Name | Hex Code | Usage |
|---|---|---|
| **Deep Space (Background)** | `#0B0F19` | Primary background for websites and IDE themes |
| **Sapphire Blue (Primary)** | `#0070F3` | Primary action buttons, primary logo gradient |
| **Neon Cyan (Accent)** | `#00F0FF` | Highlights, hover states, trust score (Safe) |
| **Alert Red (Danger)** | `#FF003C` | Blocked URLs, critical threats, error states |
| **Warning Yellow (Warn)** | `#FFB800` | Suspicious URLs, warnings |
| **Text Primary (Light)** | `#F8F9FA` | Main body text on dark themes |
| **Text Secondary (Dim)** | `#8B949E` | Muted text, timestamps, minor labels |

## 4. Typography

We use modern, highly legible sans-serif fonts optimized for code and technical documentation.

- **Headings**: `Inter` (Weights: 600, 700, 800)
- **Body Text**: `Inter` (Weights: 400, 500)
- **Monospace / Code**: `JetBrains Mono` or `Fira Code`

## 5. Voice & Tone

- **Authoritative but Humble**: We state facts clearly based on evidence. We don't use hyperbole.
- **Developer-First**: We assume our audience understands code. We focus on DX, bundle size, and latency.
- **Action-Oriented**: Instead of "This might be bad," we say "Blocked due to brand impersonation."

## 6. Naming Conventions

- The package name is always lowercase with hyphens: `safe-link-checker`.
- The architecture/platform is capitalized: `Safe Link Engine`, `Safe Link Cloud`.
- Acronyms: We avoid them where possible. Do not abbreviate the project to "SLC" in official marketing.
